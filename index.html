Introducción

Este manual describe el proceso completo para registrar masivamente estudiantes en Apache Superset al inicio de cada semestre académico. El proceso crea automáticamente los usuarios, les asigna un rol por curso y otorga los permisos necesarios para acceder al laboratorio virtual de análisis de datos.

No requiere conocimientos de programación ni acceso al servidor. Solo acceso de administrador a Superset vía navegador.

1. Preparar el archivo CSV

El archivo CSV es la fuente de datos para la carga. Debe ser preparado por el docente o coordinador académico antes de iniciar el proceso en Superset.

Estructura del archivo

El archivo debe tener exactamente las siguientes 7 columnas en el orden indicado:

ColumnaDescripciónEjemploObligatorio
first_nameNombre(s) del estudianteJuan CarlosSí
last_nameApellido(s) del estudiantePérez GómezSí
usernameCódigo o usuario único (sin espacios ni tildes)jcperez2025Sí
emailCorreo institucional del estudiantejcperez@ufps.edu.coSí
cursoNombre del programa o curso (sin espacios, usar guion bajo)Analitica_de_datosSí
yearAño académico (4 dígitos)2026Sí
semestreSemestre: 1 o 21Sí

Ejemplo del CSV

Así debe lucir el archivo al abrirlo en un editor de texto:

first_name,last_name,username,email,curso,year,semestre Juan,Pérez,jperez,jperez@ufps.edu.co,Analitica_de_datos,2026,1 Ana,Gómez,agomez,agomez@ufps.edu.co,Analitica_de_datos,2026,1 Carlos,Ruiz,cruiz,cruiz@ufps.edu.co,base_de_datos,2026,1 María,López,mlopez,mlopez@ufps.edu.co,desarrollo_web,2026,2

Reglas importantes

  • El archivo debe guardarse con codificación UTF-8 para soportar tildes y caracteres especiales en los nombres.
  • El campo username no debe contener espacios, tildes ni caracteres especiales. Use solo letras, números y guion bajo.
  • El campo curso no debe contener espacios. Use guion bajo en lugar de espacios: Ingenieria_de_Sistemas, no Ingenieria de Sistemas.
  • El campo semestre solo acepta los valores 1 o 2.
  • El campo email debe ser único por estudiante. Si un email ya existe en el sistema, ese estudiante será omitido automáticamente.
  • Si un estudiante aparece en dos cursos diferentes, debe aparecer en dos filas distintas del CSV.
El nombre del rol en Superset se construye automáticamente uniendo curso + year + semestre con guion bajo. Por ejemplo: Ingenieria_de_Sistemas_2025_1

Cómo crear el CSV desde Excel

  1. Abra Microsoft Excel y cree las columnas con los encabezados exactos: first_name, last_name, username, email, curso, year, semestre.
  2. Llene los datos de los estudiantes fila por fila.
  3. Al guardar, seleccione Archivo → Guardar como.
  4. En Tipo de archivo, seleccione CSV UTF-8 (delimitado por comas).
  5. Haga clic en Guardar. Si Excel muestra una advertencia de compatibilidad, haga clic en Continuar.
Descargar plantilla CSV de ejemplo Google Drive — listo para llenar con los datos del curso
En LibreOffice Calc: use Archivo → Guardar como → Tipo: Texto CSV. En el diálogo de exportación, seleccione separador: coma (,) y codificación: UTF-8.

2. Subir el CSV a Superset (CSV Upload)

Una vez preparado el CSV, el administrador debe cargarlo en Superset como tabla temporal. Esta tabla servirá de fuente para el SQL de carga del siguiente paso.

Este paso requiere que la opción Allow CSV Upload esté habilitada en la conexión de base de datos de Superset. Si no aparece el menú de carga, contacte al administrador del servidor.
Paso 1 — Ingresar a Superset con cuenta de administrador
  • Abra el navegador e ingrese a la URL de Superset (por ejemplo: https://superset.applab.ufps.edu.co/).
  • Ingrese con sus credenciales de administrador.
Paso 2 — Abrir el menú de carga de CSV
  • En la barra de navegación superior, haga clic en el menú Data (Datos).
  • Seleccione la opción Upload a CSV (Cargar CSV).
Ruta: Barra superior → Data → Upload a CSV
Menú Data → Upload CSV to database
Paso 3 — Configurar la carga del archivo

Complete el formulario de carga con los siguientes valores:

Campo del formularioValor a ingresar
CSV FileSeleccione el archivo estudiantes_carga.csv desde su computador
Table Nameestudiantes_carga
Databasemysql (la base de datos de Superset)
Schemasuperset
If Table Already ExistsAppend — agrega los nuevos registros sin borrar cargas anteriores
Header RowDejar en 0 (la primera fila es el encabezado)
DelimiterComa (,)
Es importante seleccionar Schema: superset. Si selecciona otro schema, el SQL del siguiente paso no encontrará la tabla.
Con Append, los registros de semestres anteriores permanecen en la tabla estudiantes_carga. Esto no genera usuarios duplicados (el SQL del paso 3 los omite automáticamente), pero si desea limpiar la tabla antes de una carga nueva, use la opción Replace o ejecute DROP TABLE estudiantes_carga; antes de subir el CSV.
Paso 4 — Ejecutar la carga
  • Revise que todos los campos estén correctos.
  • Haga clic en el botón Save (Guardar).
  • Espere a que Superset procese el archivo. Cuando termine, mostrará un mensaje de confirmación.
Si la carga fue exitosa, verá el mensaje: CSV file "estudiantes_carga.csv" uploaded to table "estudiantes_carga"

3. Ejecutar el SQL de carga en SQL Lab

Con la tabla estudiantes_carga ya cargada, ejecute el SQL de carga semestral en SQL Lab. Este proceso creará los roles por curso, copiará los permisos y registrará todos los usuarios automáticamente.

Script SQL completo (Google Drive) Incluye todos los pasos de este apartado en un solo archivo

A continuación, el paso a paso si desea ejecutarlo manualmente o entender cada bloque:

Paso 1 — Abrir SQL Lab
  • En la barra de navegación superior, haga clic en SQL.
  • Seleccione SQL Lab.
  • Se abrirá el editor de consultas SQL.
Ruta: Barra superior → SQL → SQL Lab
SQL Lab — schema superset, tabla estudiantes_carga visible
Paso 2 — Seleccionar la base de datos correcta
  • En el panel izquierdo de SQL Lab, en el selector DATABASE, elija: mysql.
  • En el selector SCHEMA, elija: superset.
  • Verifique que en SEE TABLE SCHEMA pueda buscar y ver la tabla estudiantes_carga.
Si no ve la tabla estudiantes_carga en el schema superset, regrese al paso anterior y verifique que el CSV fue cargado en el schema correcto.
Paso 3 — Verificar los datos cargados (SQL 1)

Antes de ejecutar la carga real, verifique que los datos del CSV están correctos. Copie y ejecute:

SELECT first_name, last_name, username, email, CONCAT(curso, '_', year, '_', semestre) AS rol_calculado FROM estudiantes_carga ORDER BY rol_calculado;

Revise que los roles calculados en la columna rol_calculado tienen el formato esperado, por ejemplo: Ingenieria_de_Sistemas_2025_1.

Si los datos se ven correctos, continúe con el siguiente paso.
Paso 4 — Crear los roles por curso (SQL 2)

Este SQL crea en Superset un rol por cada combinación única de curso, año y semestre. Si el rol ya existe (de un semestre anterior), lo omite automáticamente.

INSERT INTO ab_role (name) SELECT DISTINCT CONCAT(curso, '_', year, '_', semestre) FROM estudiantes_carga WHERE CONCAT(curso, '_', year, '_', semestre) NOT IN ( SELECT name FROM ab_role );
El resultado indicará cuántas filas fueron insertadas. Una fila = un rol nuevo creado.
Paso 5 — Copiar permisos de Gamma a los roles nuevos (SQL 3)

Este es el paso más importante. Copia los 81 permisos del rol Gamma (el rol estándar de estudiante) a cada rol de curso recién creado. Sin este paso, los estudiantes no podrán ver nada en Superset.

INSERT INTO ab_permission_view_role (permission_view_id, role_id) SELECT pvr.permission_view_id, r.id FROM ab_permission_view_role pvr CROSS JOIN ab_role r WHERE pvr.role_id = 4 AND r.name IN ( SELECT DISTINCT CONCAT(curso, '_', year, '_', semestre) FROM estudiantes_carga ) AND NOT EXISTS ( SELECT 1 FROM ab_permission_view_role pvr2 WHERE pvr2.permission_view_id = pvr.permission_view_id AND pvr2.role_id = r.id );
El resultado debe mostrar 81 filas insertadas por cada rol nuevo creado. Ejemplo: si creó 3 roles, deben ser 243 filas en total.
Paso 6 — Insertar los usuarios (SQL 4)

Crea los usuarios en Superset con contraseña inicial 1234. Si un username o email ya existe en el sistema, esa fila es omitida automáticamente sin generar error.

INSERT INTO ab_user (first_name, last_name, username, email, password, active, created_on, changed_on) SELECT e.first_name, e.last_name, e.username, e.email, 'scrypt:32768:8:1$35da1KJalybajFoV$d0d866761e6772fc82d061595191eaea5c62dbb589e783286fcd33c72af6274e210769a8e5b188dbb30283399b1dcff2d614b9599cfb98770580a259eba3ee36', 1, NOW(), NOW() FROM estudiantes_carga e WHERE e.username NOT IN (SELECT username FROM ab_user) AND e.email NOT IN (SELECT email FROM ab_user);
El valor largo que empieza con scrypt: es el hash seguro de la contraseña 1234. No lo modifique.
El número de filas insertadas debe coincidir con el número de estudiantes nuevos en el CSV.
Paso 7 — Asignar roles a los usuarios (SQL 5)

Asocia cada usuario al rol de su curso correspondiente.

INSERT INTO ab_user_role (user_id, role_id) SELECT u.id, r.id FROM estudiantes_carga e JOIN ab_user u ON u.username = e.username JOIN ab_role r ON r.name = CONCAT(e.curso, '_', e.year, '_', e.semestre) WHERE NOT EXISTS ( SELECT 1 FROM ab_user_role ur WHERE ur.user_id = u.id AND ur.role_id = r.id );
El número de filas insertadas debe coincidir con el total de estudiantes cargados.

4. Verificar el resultado

Ejecute la consulta de verificación final para confirmar que todos los usuarios fueron creados correctamente con su rol y permisos asignados.

Paso 1 — Ejecutar consulta de verificación (SQL 6)
SELECT u.username, u.first_name, u.last_name, u.email, r.name AS rol, (SELECT COUNT(*) FROM ab_permission_view_role WHERE role_id = r.id) AS permisos, u.active FROM ab_user u JOIN ab_user_role ur ON u.id = ur.user_id JOIN ab_role r ON r.id = ur.role_id WHERE r.name IN ( SELECT DISTINCT CONCAT(curso, '_', year, '_', semestre) FROM estudiantes_carga ) ORDER BY r.name, u.username;
Paso 2 — Validar los resultados

Verifique que en los resultados:

ColumnaValor esperado
usernameEl código de cada estudiante
rolNombre del curso con formato Curso_Year_Semestre
permisos81 — si es diferente, los permisos no se copiaron correctamente
active1 — el usuario está habilitado
Si todos los usuarios muestran permisos = 81 y active = 1, la carga fue exitosa.
Si algún usuario tiene permisos = 0, repita el SQL del paso 5 (copiar permisos de Gamma) asegurándose de que role_id = 4 corresponde a Gamma en su instancia.
Paso 3 — Verificar login de un estudiante
  • Abra una ventana de incógnito en el navegador.
  • Ingrese a la URL de Superset.
  • Use el username y la contraseña 1234 de uno de los estudiantes cargados.
  • Confirme que puede iniciar sesión y que ve los dashboards correspondientes a su curso.

5. Limpieza opcional

Una vez verificada la carga, puede eliminar la tabla temporal estudiantes_carga para mantener el schema limpio. Esta acción es opcional pero recomendada.

DROP TABLE estudiantes_carga;
Si necesita verificar la carga nuevamente en el futuro, deberá subir el CSV otra vez. Se recomienda guardar el archivo CSV en un repositorio o carpeta compartida del equipo.

6. Resolución de problemas comunes

ProblemaCausa probableSolución
Error al hacer CSV Upload: "This database does not allow DDL/DML" La conexión de BD no tiene habilitado Allow DML Settings → Database Connections → Editar mysql → Advanced → SQL Lab → Activar "Allow DML"
No aparece la tabla estudiantes_carga en SQL Lab El CSV fue cargado en un schema diferente Repetir CSV Upload seleccionando Schema: superset
Los usuarios fueron creados pero la columna permisos muestra 0 El SQL de copiar permisos no se ejecutó o falló Volver a ejecutar el SQL 3 (Paso 5 de la sección 3)
El login del estudiante falla con 1234 El hash de la contraseña se modificó o copió incorrectamente Verificar que el valor scrypt: en el SQL 4 es exactamente el indicado en este manual
Algunos estudiantes no fueron creados Su username o email ya existe en el sistema Ejecutar: SELECT username, email FROM ab_user para ver usuarios existentes
El rol creado tiene un nombre incorrecto El campo curso en el CSV tenía espacios o caracteres especiales Corregir el CSV reemplazando espacios por guion bajo y volver a cargar
